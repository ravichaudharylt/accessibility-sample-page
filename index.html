Repro — unique-selector crash on malformed element IDs

How to reproduce:
  1. Enable AI accessibility scan in the extension settings (feature flag accessibility_ai_issues)
  2. Open this page in Chrome with the LambdaTest Accessibility extension loaded
  3. Run a full-page scan
  4. Scan will fail with: Failed to execute 'querySelectorAll' on 'Element': '#Due TodaySection 2 Receipt Update' is not a valid selector.
Crash path:
capturePageData() → extractElements() walks DOM with TreeWalker → finds img/link/svg elements → calls unique(node) from unique-selector library → unique() walks parent chain via getParents(el) → hits ancestor with id="Due TodaySection 2 Receipt Update" → getID() returns #Due TodaySection 2 Receipt Update (only escapes IDs starting with digits or containing :, not spaces) → testUniqueness() calls parentNode.querySelectorAll('#Due TodaySection 2 Receipt Update') → throws DOMException → bubbles through Promise.all into catch block → SCAN_RUN_FAILED

Notes

  • The crash does not come from axe-core — axe v4.11.0 correctly escapes IDs with CSS.escape().
  • The crash comes from the unique-selector npm package used in extractPageData.js.
  • capturePageData runs whenever the accessibility_ai_issues feature flag is enabled — the capture screenshot setting has no effect.
  • Elements with invalid IDs must be in the light DOM (not shadow DOM) for the TreeWalker to reach them.
  • The invalid-ID element itself doesn't need to be a scannable type — it just needs to be an ancestor of an img, link, or SVG.
